# ADR-0004: Use quick-xml for S3 XML serialization

## Status
Accepted

## Context
The S3 API uses XML for several request and response bodies:
- `CreateBucketConfiguration` with `LocationConstraint` (outgoing)
- `ListAllMyBucketsResult` with nested `Buckets > Bucket` elements (incoming)
- `ListBucketResult` with repeated `Contents` elements (incoming)

We need a library that can serialize Rust structs to XML and deserialize XML responses into Rust structs.

## Decision
Use **quick-xml** with the `serialize` feature for S3 XML handling.

## Alternatives considered
- **xml-rs**: Verbose, event-based API. No serde integration. Would require manual XML construction.
- **serde-xml-rs**: Less maintained than quick-xml. Slower development cadence.
- **yaserde**: Smaller ecosystem. Less battle-tested.
- **Manual string formatting**: Not scalable as more XML types are added. Error-prone for escaping.

## Consequences
Positive:
- Serde derive integration: `#[derive(Serialize, Deserialize)]` with `#[serde(rename)]` handles XML element mapping
- Handles repeated elements (Vec<T>) and nested structures naturally
- Same library used by awrust-s3 server — consistent XML handling across the ecosystem

Negative:
- Serde-based XML has quirks with namespaces and attribute handling
- The `serialize` feature increases compile time
