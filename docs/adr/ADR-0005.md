# ADR-0005: Use Serde as the serialization framework

## Status
Accepted

## Context
awrust-cli needs to serialize and deserialize data in XML format for S3 API interactions. The XML library choice (ADR-0004) uses serde as its abstraction layer.

We want:
- A single serialization framework that works across formats
- Derive macros for minimal boilerplate
- Field renaming support (Rust snake_case to XML PascalCase)

## Decision
Use **Serde** with the `derive` feature as the serialization framework.

## Alternatives considered
- **Manual serialization**: Duplicates effort and is error-prone for each new XML type.
- **Format-specific traits**: Would require different serialization code per format, violating the principle of extracting when patterns emerge.

## Consequences
Positive:
- `#[derive(Serialize, Deserialize)]` with `#[serde(rename)]` handles all XML type mapping
- If JSON output is added later, the same types work with serde_json
- Consistent with awrust ecosystem conventions (ADR-0006 in awrust-s3)

Negative:
- Foundational dependency that increases compile time
- Proc macro expansion adds to binary size in debug builds
