# ADR-0002: Use Clap for CLI argument parsing

## Status
Accepted

## Context
awrust-cli needs to parse a nested command structure (`awr s3 mb`, `awr s3 ls`, etc.) with global flags (`--endpoint`), subcommand-specific arguments, and environment variable fallback (`AWRUST_ENDPOINT`).

We want:
- Derive-based argument definitions for compile-time verification
- Nested subcommand support
- Built-in environment variable fallback
- Help text generation

## Decision
Use **Clap** with the `derive` and `env` features for CLI argument parsing.

## Alternatives considered
- **structopt**: Merged into Clap 4; using Clap directly gives access to the latest features.
- **argh**: Simpler but lacks environment variable support and has a smaller ecosystem.
- **Manual parsing**: Error-prone and requires reimplementing help, validation, and env var support.
- **bpaf**: Newer, less proven; smaller community.

## Consequences
Positive:
- Subcommand hierarchy (`Command::S3 { action: S3Command }`) maps directly to the CLI surface
- `#[arg(env = "AWRUST_ENDPOINT")]` handles env var fallback with zero custom code
- Compile-time verified: missing fields or wrong types are caught at build time
- Auto-generated `--help` for every command and subcommand

Negative:
- Significant compile-time cost (proc macro + derive expansion)
- Large dependency tree for a CLI parser
